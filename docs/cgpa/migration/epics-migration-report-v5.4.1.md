# Rapport de migration des Epics — CGPA v5.4.1

Les Epics, lots post-go-live et périmètres de release sont conservés sans renumérotation ni
requalification rétroactive. La migration v5.4.1 ne modifie aucun statut fonctionnel.

Toute future promotion d'un Epic, d'une Release ou d'un Hotfix vers Production doit vérifier que
le Staging source dispose d'une preuve `STG-ISOL-01` conforme à l'ADR canonique
`docs/cgpa/adr/ADR-STG-001-staging-isolation.md`.

Décision : **GO sous réserve** des risques Staging ouverts et des réserves Release Management
déjà enregistrées.
