# Checklist documentation

> À dérouler en continu, en particulier en phase **08 (développement)** et avant chaque gate.

## 1. Documentation projet
- [ ] README à jour (présentation, installation, usage)
- [ ] Livrables CGPA présents et à jour (EB, faisabilité, CDC, architecture)
- [ ] Décisions d'architecture consignées (ADR)

## 2. Documentation technique
- [ ] Code commenté là où c'est nécessaire (intentions, choix non triviaux)
- [ ] API documentée (contrats, exemples)
- [ ] Modèle de données documenté
- [ ] Variables d'environnement / configuration documentées

## 3. Documentation d'exploitation
- [ ] Procédure de déploiement
- [ ] Procédure de rollback / restauration
- [ ] Runbook incidents (supervision, alertes)
- [ ] Procédure de sauvegarde

## 4. Documentation utilisateur (si applicable)
- [ ] Guide d'utilisation / onboarding
- [ ] FAQ / dépannage

## 5. Traçabilité & gouvernance
- [ ] Gates consignés (décisions, scores, dates)
- [ ] Backlog et statut des sprints à jour
- [ ] Bilan post-production (en fin de cycle)

## 6. Qualité documentaire
- [ ] Documentation versionnée avec le code
- [ ] Langue et format cohérents
- [ ] Pas d'information obsolète ou contradictoire
