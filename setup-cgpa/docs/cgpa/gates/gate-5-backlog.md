# Gate 5 — Backlog & planification Agile

> Frontière entre la phase **06 (Planification Agile)** et la phase **07 (DevSecOps)**.

## Conditions d'entrée
- Gate 4 statué Go (autorisation de coder accordée).
- Architecture validée.

## Livrables obligatoires
- [ ] `templates/product-backlog.md` complété.
- [ ] `templates/sprint-planning.md` (sprint 1) complété.

## Critères de validation
- [ ] Backlog décomposé en epics et user stories (INVEST).
- [ ] Critères d'acceptation pour chaque story.
- [ ] Priorisation par valeur (MoSCoW/WSJF).
- [ ] Definition of Done incluant tests + sécurité + doc.
- [ ] Premier sprint planifié selon la capacité réelle.
- [ ] Score ≥ 8/20.

## Risques bloquants
- Stories sans critères d'acceptation.
- Backlog non priorisé par la valeur.
- Capacité non prise en compte (sur-engagement).
